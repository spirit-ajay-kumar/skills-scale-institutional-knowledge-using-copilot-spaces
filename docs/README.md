# OctoAcme Project Management Docs

Welcome to the central hub for OctoAcme’s project management processes and templates. This documentation ensures consistent execution, supports onboarding, and helps capture critical team knowledge so every member can confidently contribute to projects from day one.

---

## Project Management Process Overview

OctoAcme uses a structured, iterative approach for managing projects:

- **Customer-first:** Prioritize outcomes that deliver measurable value.
- **Iterative delivery:** Break work into small, testable increments.
- **Clear ownership:** Every project has a named Project Manager (PM) and Product Lead.
- **Continuous improvement:** Reflect after each cycle, capture learnings, and refine processes.
- **Transparent documentation:** Maintain standardized artifacts, templates, and reporting.
- **Data-informed decisions:** Use metrics and success criteria to guide choices.
- **Psychological safety:** Enable candid feedback, learning, and adaptation.

**Project Lifecycle Stages:**

1. **Initiation:** Validate business need, identify stakeholders, define outcomes, create a one-pager.
2. **Planning:** Break down work, estimate scope, identify risks/dependencies, create a release plan.
3. **Execution & Tracking:** Deliver, measure progress, hold standups and syncs, report on metrics.
4. **Risk Management & Communication:** Capture and address risks, communicate status and issues promptly.
5. **Release & Deployment:** Use repeatable, safe processes for production releases.
6. **Retrospective & Continuous Improvement:** Run retrospectives, assign actions, and improve iteratively.

**Core Roles:**  
Project Manager (PM), Product Manager, Developers, QA/Testing, Stakeholders — see [Roles and Personas](./octoacme-roles-and-personas.md).

**Artifacts:**  
Project One-pager, Backlog, Release Plan, Risk Register, Retro notes, status updates, definition of done, and more.
